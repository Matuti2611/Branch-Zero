# Branch-Zero

La instrucción Branch Zero (BRZ), o salto condicional si es cero, es una operación de control de flujo que permite la ejecución no lineal de un programa. Su mecanismo subyacente depende estrechamente del registro de estado del procesador, específicamente del indicador de condición conocido como Zero Flag (ZF). Tras la ejecución de una instrucción aritmética o lógica por parte de la Unidad Aritmético Lógica (ALU), como puede ser la sustracción comparativa de dos operandos, el hardware evalúa el resultado; si este es exactamente nulo, el ZF se activa estableciéndose en un valor lógico alto. Cuando la unidad de control decodifica una instrucción BRZ, interroga inmediatamente este indicador para determinar el curso de acción: si el ZF se encuentra activo, la condición preestablecida se considera verdadera y se habilita el desvío del flujo. Si la condición de cero no se cumple (ZF inactivo), la lógica de control simplemente incrementa el PC para apuntar a la instrucción secuencial contigua, ignorando el salto. Sin embargo, si la condición se satisface, el hardware de ruteo inyecta la dirección de destino, típicamente calculada mediante un desplazamiento o offset codificado en la propia instrucción, directamente en el PC a través de un multiplexor.

Para superar la limitación de entradas del simulador, la detección de cero de 16 bits se implementa de forma segmentada. El bus de datos se divide en dos bloques de 8 bits, evaluados concurrentemente por dos compuertas NOR. Las salidas de ambas convergen en una primera compuerta AND para sintetizar la bandera de cero (Zero Flag), confirmando que toda la palabra de 16 bits es nula. Por último, una segunda compuerta AND evalúa este indicador junto con el opcode, garantizando que la señal de salto solo se emita si el resultado es cero y la instrucción en curso es efectivamente un Branch Zero.
